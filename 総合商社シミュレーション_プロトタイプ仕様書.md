# 総合商社の仕事の面白さを体験できるシミュレーションゲーム 仕様書

## A. ゲームの核となるコンセプト要約

### 1. ゲームの一文コンセプト
世界中に散らばる「需要」「供給」「資金」「物流」「制度」「人脈」の断片をつなぎ、単発取引ではなく持続的な収益構造を設計する総合商社シミュレーション。

### 2. プレイヤーが味わうコア体験
- 情報の断片から「まだ名前のない案件」を見抜く
- 条件の足りない案件を、パートナー・契約・資金・物流で成立させる
- 社内外の利害を調整し、案件を稟議通過まで持っていく
- 投資した事業を運営し、危機時に救済・再編して価値を守る
- 単独案件ではなく、ポートフォリオ全体でシナジーを作る

### 3. 快感の正体
- 発見の快感: バラバラの市場情報が一本の「事業仮説」に変わる
- 組成の快感: 足りない条件を埋めて案件成立率が上がる
- 交渉の快感: 相手ごとの欲しいものを読み、通る提案に変える
- 構造化の快感: 単発利益よりも、次の案件も生む基盤が育つ
- 逆転の快感: 危機で終わりそうな案件を、他事業や契約再編で救う

### 4. 総合商社らしさを表現する核
- 案件はマップに完成品として落ちておらず、複数の断片情報を集めて成立要件を埋める
- 収益源が「トレード差益」だけでなく、「持分利益」「利用料」「JV配当」「物流マージン」「周辺事業波及」に広がる
- 政府、銀行、メーカー、現地企業、本社、ESG部門など相反する主体を同時に動かす
- 個別案件の採算だけでなく、全社の地域分散、産業分散、資源配分、資本効率、リスク耐性で最適化する
- バリューチェーンを保有・接続することで、次の案件が取りやすくなり危機にも強くなる

### 5. 他の経営ゲーム・貿易ゲームとの違い
- 貿易ゲームとの差: 「安く買って高く売る」より、「成立しない案件を成立させる」ことが中心
- 工場経営ゲームとの差: 自社設備最適化より、他社資産・制度・資金を束ねることが中心
- 国際政治ゲームとの差: 国家運営ではなく、民間主体として事業を通じて構造を変える
- カードゲームとの差: ランダム運ではなく、情報収集・交渉・構造設計の積み重ねが勝ち筋になる

### 6. MVP方針の提案
- 基本軸はユーザー案を採用: 地域は日本、東南アジア、中東。産業はエネルギー、食料、モビリティ、インフラ
- ただし初回MVPは「10ターン」より「8ターン」を推奨
理由: 1ターンを濃く設計するため、最初は 60-90 分で1周できる方がテストしやすい
- 1ターン=1年は維持してよい
- 代わりに毎ターンの行動回数を制限し、「何を深掘りし、何を捨てるか」の意思決定を作る

## B. 1ターンのゲームループ

### 1. 全体進行
- 想定プレイ期間: 8ターン
- 1ターン: 1年
- 1ターンあたりのプレイヤー資源:
  - 経営アクション 6
  - 専門人材アサイン枠 8
  - 稟議提出数 2
  - 危機対応即応枠 1

### 2. フェーズ構成

#### フェーズ1: 情勢変化フェーズ
- 表示:
  - 地域別マクロ指標
  - 資源価格
  - 為替
  - 政策ニュース
  - リスク警報
- プレイヤー判断:
  - どの地域・産業が追い風か
  - どの既存事業に警戒が必要か
- 変化:
  - 市況、資金コスト、規制、需要成長率、イベント発生率が更新される

#### フェーズ2: 情報収集フェーズ
- 表示:
  - 地域 x 産業ヒートマップ
  - 情報源候補
    - 政府
    - 銀行
    - メーカー
    - 物流会社
    - 現地財閥
    - 業界紙
- プレイヤー判断:
  - どの地域/産業に人材を張るか
  - 広く浅く拾うか、仮説案件を深掘りするか
- 変化:
  - 「断片情報カード」が獲得される
  - 情報の確度、鮮度、公開期限が付与される

#### フェーズ3: 案件発見フェーズ
- 表示:
  - 集めた断片情報一覧
  - 新規に開いた「案件リード」
  - 既存リードの不足条件
- プレイヤー判断:
  - どのリードを案件化するか
  - どの断片を組み合わせて仮説を立てるか
- 変化:
  - 断片のタグが一定条件を満たすと「案件キャンバス」が開放される

#### フェーズ4: 案件組成フェーズ
- 表示:
  - 案件テンプレート
  - 必須スロット
    - 需要
    - 供給/技術
    - 現地基盤
    - 資金
    - 許認可
    - 物流
    - 販売先/オフテイク
    - 運営主体
  - 採算試算
  - リスク露出
  - シナジー期待値
- プレイヤー判断:
  - どのパートナーで各スロットを埋めるか
  - どこを自社で持ち、どこを外部化するか
  - 負債比率、契約形態、JV比率、出口戦略をどうするか
- 変化:
  - 案件の成立可能性、想定IRR、稟議難易度、必要交渉先が確定する

#### フェーズ5: 交渉フェーズ
- 表示:
  - 相手の関心事項
  - 信頼度
  - BATNA（代替案の強さ）
  - 提案内容
  - 想定受諾率
- プレイヤー判断:
  - 何を譲り、何を守るか
  - 誰から先に口説くか
  - 今期の利益と長期の支配権のどちらを取るか
- 変化:
  - 各ステークホルダーの協力度
  - 契約条件
  - 関係値
  - 情報開示量

#### フェーズ6: 社内稟議フェーズ
- 表示:
  - 財務
  - リスク管理
  - 法務
  - ESG
  - 経営陣
  - 事業本部
  の評価メーター
- プレイヤー判断:
  - 稟議を出す案件を2件まで選ぶ
  - 追加の根回しを行うか
  - 条件修正して再提出するか
- 変化:
  - 稟議承認、条件付き承認、差し戻し、否決
  - 社内評価の増減
  - 次ターンの資本配分余力

#### フェーズ7: 実行・運営フェーズ
- 表示:
  - 建設進捗
  - 稼働率
  - 契約履行状況
  - キャッシュフロー
  - 現地評判
- プレイヤー判断:
  - 追加投資するか
  - 人材を再配置するか
  - 低採算案件から撤退するか
- 変化:
  - 稼働中案件が収益と信頼を生む
  - 建設中案件は進捗し、遅延・コスト超過の判定が入る

#### フェーズ8: リスク発生・対応フェーズ
- 表示:
  - 発生イベント
  - 影響を受ける案件
  - 対応オプション
  - 想定損失/救済確率
- プレイヤー判断:
  - 損切り
  - 契約再編
  - 追加投資
  - 他地域・他事業への切替
  - 現地関係者への火消し
- 変化:
  - 損益、信頼、ESG、将来の機会開放が変わる

#### フェーズ9: 決算・ポートフォリオ評価フェーズ
- 表示:
  - 売上
  - 営業CF
  - 投資残高
  - 持分利益
  - ROIC
  - 地域集中度
  - 産業分散
  - リスク耐性
  - シナジー数
- プレイヤー判断:
  - 来期は短期回収を優先するか
  - 次の柱作りに再投資するか
- 変化:
  - 年度評価
  - 次ターンの信用枠
  - 取締役会からの期待値

### 3. ゲーム全体の進行テンション
- 1-2ターン目: 情報網作りと初期案件の布石
- 3-5ターン目: 中核案件の組成と初回投資
- 6-7ターン目: 危機対応とシナジー回収
- 8ターン目: 企業価値最大化の仕上げ

## C. コアシステム一覧

| システム | ゲーム内の役割 | プレイヤーの悩み | 実装の要点 |
| --- | --- | --- | --- |
| 断片情報システム | 世界の兆候を案件の種に変える | 何を追うか、何を捨てるか | タグ、確度、期限、出所を持つカード |
| 案件キャンバス | 断片から案件を組み立てる | どのスロットを誰で埋めるか | 必須スロット制、空欄があると不成立 |
| パートナー選定 | 成立率と採算を変える | 条件が良い相手か、信頼できる相手か | 能力タグ、交渉重視項目、関係値を保持 |
| 交渉システム | 相手の利害を動かす | 支配権、利益率、スピードのどれを取るか | 提案パッケージ方式 |
| 社内稟議システム | 商社らしい意思決定を表現 | 良案件でも今通すべきか | 部門別スコアと veto 条件 |
| 投資・運営システム | 事業化後の重みを作る | 稼働改善に人材を割くか次案件か | 建設/運営/再編の状態管理 |
| リスクイベントシステム | 市況変動と危機対応を生む | 防御するか攻めるか | 露出タグ + 発生条件 + 対応分岐 |
| シナジーグラフ | 商社の「つなぐ力」を表現 | 単独採算より構造価値を取るか | 保有アセット間の接続判定 |
| ポートフォリオ管理 | 全社最適を作る | 収益、分散、将来種まきの両立 | 集中リスクと option value の両方を可視化 |
| 評価システム | 勝敗を定義する | 短期利益偏重を防ぐ | 企業価値、耐性、戦略資産で総合評価 |

### 1. 勝敗条件・評価指標
最終スコアは単年利益ではなく、以下の総合値とする。

`企業価値スコア = 累積キャッシュ創出 + 保有事業NPV + シナジー価値 + 信用プレミアム - 危機損失 - 集中リスク罰則`

#### 評価軸
- 収益力: 累積営業CF、持分利益、ROIC
- 成長性: 新規案件成立数、将来収益化予定案件数
- 構造価値: シナジー接続数、バリューチェーン完成数
- 耐性: 地域分散、産業分散、リスク対応成功率
- 社会的持続性: ESG評価、現地信頼、重大炎上の少なさ

### 2. 正解の固定化を防ぐ仕組み
- 市況とイベントの組み合わせで毎回価値の高い産業が変わる
- 断片情報の出方で案件構造が変わる
- 相手の優先項目が毎回微変動する
- 社内の評価軸も年ごとに変動する
  - 例: 今期は財務重視、次期はESG重視

### 1. 人材の内訳
- オリジネーション: 情報収集、案件発見に強い
- 事業開発: 案件組成、パートナー選定に強い
- 財務/プロジェクトファイナンス: 借入条件、稟議説得に強い
- 法務/契約: 規制、契約再編、危機対応に強い
- オペレーション: 稼働改善、事故抑制、立ち上げに強い
- 渉外/政府関係: 許認可、現地信頼、政府交渉に強い

### 2. リソース間の相互作用
- 社内評価が高いほど投資余力が増える
- ESG評価が低いと、銀行条件悪化 + 社内ESG veto の危険が上がる
- 現地信頼が高いと危機時に「救済オプション」が追加される
- 保有アセットが多いと情報網が自然成長する

## E. 案件・交渉・リスクの設計詳細

### E-1. 案件システム詳細

#### 1. 案件の基本状態
- リード: まだ仮説段階
- 組成中: スロットを埋めている段階
- 稟議待ち: 外部条件はほぼ固まり、社内承認待ち
- 実行中: 建設・立ち上げ中
- 稼働中: 収益化済み
- 再編中: 問題案件の立て直し中
- 売却済み/撤退済み: 終了

#### 3. 案件テンプレート方式
案件は1件ずつ完全ランダム生成するより、以下の「テンプレート + 断片情報 + 地域条件」で組み立てる。

##### テンプレート例
- エネルギー:
  - LNG to Power
  - 再エネ + 系統接続
  - 送配電設備更新
- 食料:
  - 穀物輸入 + サイロ + コールドチェーン
  - 飼料供給 + 畜産加工
- モビリティ:
  - EV輸入販売 + 充電網
  - 商用車リース + 保守金融
- インフラ:
  - 港湾運営 + 背後物流
  - 工業団地 + 電力・水供給

#### 4. 「案件を探す」のではなく「組み立てる」ためのロジック
案件の発見条件は「完成案件の出現」ではなく、「成立要件の見える化」とする。

##### 断片情報のタグ例
- Need: 需要増、供給不足、老朽化、輸入依存
- Supply: 技術、原料、商品供給、製造余力
- Finance: 融資意欲、低利資金、補助金、輸出信用
- Access: 土地、港、送電網、販売網、規制緩和
- Partner: 地場企業、政府、財閥、商社、オフテイカー
- Risk: 政変懸念、為替不安、港湾脆弱、世論反発

##### 案件リード生成ルール
- 同一地域または同一回廊で、以下を満たすと案件リードが開く
  - Need 系タグを 1枚以上
  - Supply/Partner 系タグを 1枚以上
  - Finance/Access 系タグを 1枚以上
- 断片情報の確度合計が一定値を超えると「仮説案件」として可視化
- 可視化後、追加調査でテンプレートを選択できる
- テンプレート選択後、必要スロットと不足条件が表示される

##### 面白さを生むポイント
- 同じ断片でも、別テンプレートに組み替えられる
  - 例: 「港が弱い」はマイナス要素だが、港湾運営案件の入口にもなる
- 欠けている条件は、金で解決できるものと、人脈でしか解決できないものがある
- 組み合わせによって、単発案件が地域プラットフォーム案件に変わる

#### 5. 成立可能性の計算

`案件成立スコア = スロット充足 + 条件適合 + 関係値補正 + 資金成立性 + シナジー補正 - 未解決リスク`

##### スコア解釈
- 80以上: 高確率で稟議に進める
- 60-79: 条件付きで進行可能
- 40-59: 要追加調査・追加交渉
- 39以下: 構造不成立

#### 6. 投資と回収
各案件は以下のキャッシュフロープロファイルを持つ。

- トレード型:
  - 初期投資小
  - 即年収益
  - ボラティリティ高
- 事業投資型:
  - 初期投資大
  - 収益化まで 2-3 ターン
  - 稼働後は安定収益
- インフラ型:
  - 収益立ち上がり遅い
  - リスク低め
  - 現地信頼 + 将来案件開放効果が大きい
- 資源型:
  - 収益天井高い
  - 価格連動リスク大
  - シナジー価値も大きい

#### 7. 事業運営
稼働中案件は毎ターン以下を持つ。
- 売上
- EBITDA
- 稼働率
- 契約健全性
- 現地評判
- 事故率
- 改善余地

プレイヤーは人材配置で以下を調整できる。
- 稼働率改善
- コスト削減
- 炎上抑制
- 契約再交渉
- 拡張投資

### E-2. 交渉・関係者システム

#### 1. 関係者の種類
- 現地政府
- 国営企業/公社
- 現地民間企業
- 現地財閥
- 日本メーカー
- 海運/物流会社
- 銀行/ファンド
- オフテイカー
- 国際機関
- 本社財務
- 本社法務
- 本社リスク管理
- ESG部門
- 経営陣

#### 2. 関係者ごとの重視項目

| 関係者 | 重視項目 |
| --- | --- |
| 現地政府 | 雇用、税収、電力安定供給、主権、政治的説明可能性 |
| 国営企業 | 安定供給、経営権、国内調達、料金水準 |
| 現地企業 | 利益配分、経営権、早期回収、信用補完 |
| 現地財閥 | 面子、支配権、系列維持、独占権 |
| 日本メーカー | 販売数量、単価、防衛可能な利益率、長期保守契約 |
| 銀行 | DSCR、安全性、担保、契約安定性 |
| オフテイカー | 価格、供給安定、柔軟性、違約時保護 |
| 本社財務 | ROIC、回収期間、資本効率、格付影響 |
| 本社法務 | 契約保全、規制適合、紛争時の保護 |
| 本社リスク管理 | 地政学、為替、集中リスク、事故リスク |
| ESG部門 | 環境負荷、労働、地域受容性、説明責任 |
| 経営陣 | 今期利益、長期戦略、象徴案件性、再現性 |

#### 3. 交渉の基本ルール
- 1回の交渉で選べるのは「議題1つ + 提案2つ + 譲歩1つ」
- 交渉相手には hidden weight がある
- 関係値が高いほど、相手の hidden weight が一部開示される
- 交渉は最大3ラウンド
- 妥結しなければ、相手は代替候補へ移る可能性がある

#### 4. 提案項目の種類
- 出資比率
- 経営権
- 最低購入保証
- 長期売買契約
- 価格調整条項
- 技術移転
- 現地雇用比率
- ローカル調達比率
- ESG投資
- 独占権
- 融資保証
- 退出オプション

#### 5. 受諾判定

`受諾度 = 提案適合度 + 関係値 + 相手の切迫度 - 支配権喪失懸念 - ESG/法務懸念`

##### 例
- 財閥相手に利益率を高く渡しても、経営権を奪う提案だと不満
- 銀行は利回りよりも契約安定性と保証に強く反応
- 政府は金額より、雇用・税収・料金安定を見やすい

#### 6. 社内稟議システム
社内関係者も「交渉対象」として扱うのが重要。

##### 稟議評価軸
- 収益性
- 戦略整合
- 資本効率
- リスク許容内か
- ESG適合
- 説明可能性

##### 部門ごとの veto 条件例
- 財務: 回収期間が長すぎる
- リスク管理: 特定地域への集中が高すぎる
- 法務: 重要契約の保全が弱い
- ESG: 環境・人権懸念が未対処

##### 根回しアクション
- 事前説明
- 条件修正
- 保険/ヘッジ追加
- 他案件との抱き合わせ説明
- 撤退条件を明記

これにより「案件は良いのに社内で通らない」という商社らしい悩みを作る。

### E-3. シナジー設計

#### 1. シナジーは passive bonus だけにしない
シナジーは単なる数値上昇ではなく、以下を解放する。
- 新規案件の開放条件
- 危機時の救済選択肢
- 交渉優位
- 情報取得効率上昇

#### 2. アセットタグ
各案件・保有事業は以下のタグを持つ。
- 地域タグ: Japan, SEA, MiddleEast
- 産業タグ: energy, food, mobility, infrastructure
- バリューチェーンタグ: upstream, logistics, processing, utility, retail, finance
- 能力タグ: govt_relation, operator, financing, manufacturing, distribution
- ESGタグ: green, transition, high_emission, social_impact

#### 3. シナジー発生ルール

##### 縦のシナジー
- upstream + logistics + downstream が同じ商品群でつながる
- 効果:
  - 利益率 +10%
  - 調達途絶リスク -20%
  - 新規案件の成立スコア +10

##### 地域プラットフォームシナジー
- 同一地域で 2件以上の稼働資産を持つ
- 効果:
  - 情報網 +1/ターン
  - 現地信頼 +1/ターン
  - 政府交渉の初期関係値上昇

##### 機能シナジー
- financing + operator + govt_relation を保有
- 効果:
  - 大型インフラ案件の稟議難易度低下
  - 銀行交渉の受諾度上昇

##### 危機救済シナジー
- 代替物流または代替販売網を持つ
- 効果:
  - 物流寸断・輸出規制時に「切替」選択肢が追加

### E-4. リスクイベント設計

#### 1. イベントカテゴリ
- 市況変動
- 地政学
- 規制変更
- 為替
- サプライチェーン寸断
- 事故/品質問題
- ESG炎上
- 自然災害
- パートナー不祥事

#### 2. イベント設計項目

| 項目 | 内容 |
| --- | --- |
| id | イベント識別子 |
| category | カテゴリ |
| trigger | 発生条件 |
| targetScope | 地域、産業、特定タグ |
| severity | 影響度 |
| immediateEffect | 即時損失/遅延/評判低下 |
| lingeringEffect | 数ターン残る影響 |
| mitigationOptions | 回避/緩和手段 |
| upsidePath | 逆に機会化する条件 |

#### 3. イベント例

##### 例1: 資源価格暴落
- 発生条件: 世界景気後退 + 資源関連保有比率高
- 影響範囲: 資源・エネルギー案件
- 即時影響: 利益率低下、資産評価損
- 緩和策:
  - ヘッジ実行済み
  - 下流需要家との長期契約
  - 物流・発電資産との内製消化
- チャンス化:
  - 安値で追加権益取得
  - 下流案件の原料コスト改善

##### 例2: 輸出規制強化
- 発生条件: 地政学緊張上昇
- 影響範囲: モビリティ、エネルギー、先端部材
- 緩和策:
  - 代替調達国を確保
  - 現地組立化
  - 契約再編
- チャンス化:
  - 規制対象外地域での加工/物流案件が有利になる

##### 例3: 現地パートナー不祥事
- 発生条件: 関係値低い相手との拙速提携、ガバナンス不足
- 影響範囲: 当該案件 + ESG評価
- 緩和策:
  - 監査権の確保
  - 共同管理条項
  - 代替運営者への切替
- チャンス化:
  - 持分買い増しによる支配権取得

##### 例4: 大規模洪水
- 発生条件: モンスーン地域 + 気候リスク高
- 影響範囲: 港湾、物流、食料、インフラ
- 緩和策:
  - 保険
  - 冗長物流
  - 地域分散
- チャンス化:
  - 復旧インフラ案件が新規開放

#### 4. リスクをゲームとして面白くする原則
- 突発損失だけで終わらせず、必ず「対応の余地」を残す
- 対応はコストだけでなく、構造転換の入口にする
- 事前に準備した契約やアセットが危機時に意味を持つ
- 何もしていないと運ゲー感が強くなるので、露出と緩和策を可視化する

## I. このゲームを面白くするための重要原則

1. 常に「情報の断片」から始める
完成案件が湧く設計にすると、商社らしさが消える。最初に見えるのは不完全な兆候だけにする。

2. 案件は採算だけでなく、構造価値で評価させる
今期利益が低くても、次案件の扉を開く案件は強く魅力的であるべき。

3. 交渉は確率ゲームではなく、相手理解ゲームにする
サイコロより、目的関数の読み合いで結果が変わる設計にする。

4. 社内も外部も同じくらい重要にする
外で通っても社内で落ちる。逆に社内の後押しがあるから外でも通る。この往復が商社らしい。

5. リスクは罰ではなく、再設計の機会にする
良い危機対応は「損失最小化」だけでなく「構造転換」につながるべき。

6. シナジーは次の選択肢を増やす形で実感させる
数値+5%では弱い。新しい案件、救済策、交渉優位が開く形にする。

7. プレイヤーを常に板挟みにする
短期利益 vs 長期投資、支配権 vs 妥結速度、安全性 vs 成長性。この緊張が消えると退屈になる。

8. 非商社ユーザーにも理解できる形に翻訳する
専門用語は使ってよいが、必ず「何が嬉しいか」「何が危ないか」を UI で説明する。

9. 最適解を固定しない
毎回同じ産業・同じ地域が勝つと、ポートフォリオゲームとして死ぬ。

10. 1回のプレイで「自分なりの商社像」が出るようにする
資源型、インフラ型、地域密着型、トレード巧者型など、戦略人格が結果に出ると面白い。
